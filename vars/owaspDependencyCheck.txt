<?xml version="1.0" encoding="UTF-8"?>
    <p>
        The <code>owaspDependencyCheck</code> variable offers access to the OWASP Dependency Check.
    </p>
    <p>
        Example call: 
        <code>
            stage('OWASP Dependency Check') {
            steps {
                owaspDependencyCheck "app", "api", extraArgs: "--enableExperimental --suppression dependency-check-suppression.xml --failOnCVSS 7 --exclude exclude"
            }
        </code>
    </p>
    <p>
        The name of the OWASP Dependency Check in the Global Tool Configuration.
        Default: owasp-dependency-check-5.2.4
    </p>
    <p>
        The generated reports (<code>--format 'ALL'</code>) dependency-check-junit.xml, dependency-check-report.csv, dependency-check-report.html, dependency-check-report.json and dependency-check-report.xml are written to the folder report (<code>--out report</code>) in the current workspace.
    </p>
    <p>
        The first arguments are the application folders to be scanned. In the example above: <code>app</code> and <code>api</code>.
    </p>
    <p>
        Extra arguments can be added to the call with <code>extraArgs: ""</code>.
        For a full list of available arguments see: https://jeremylong.github.io/DependencyCheck/dependency-check-cli/arguments.html
        Important arguments are: 
    </p>
    <p>
        For further information see documentation on GitHub: https://jeremylong.github.io/DependencyCheck/.
    </p>
    <dl>
        <dt><code>--enableExperimental</code></dt>
        <dd>
            <p>
                Enable the experimental analyzers. If not set the analyzers marked as experimental below will not be loaded or used. 
                File Type and Experimental Analyzers: https://jeremylong.github.io/DependencyCheck/analyzers/index.html
            </p>
        </dd>
        <dt><code>--exclude <pattern></code></dt>
        <dd>
            <p>
                The path patterns to exclude from the scan - this option can be specified multiple times. This accepts Ant style path patterns (e.g. /exclude/). 
            </p>
        </dd>
        <dt><code>--failOnCVSS</code></dt>
        <dd>
            <p>
                If the score set between 0 and 10 the exit code from dependency-check will indicate if a vulnerability with a CVSS score equal to or higher was identified.
            </p>
        </dd>
        <dt><code>--suppression <files></code></dt>
        <dd>
            <p>
                The file paths to the suppression XML files; used to suppress false positives. This can be specified more than once to utilize multiple suppression files. The argument can be a local file path, a URL to a suppression file, or even a reference to a file on the class path (see https://github.com/jeremylong/DependencyCheck/issues/1878#issuecomment-487533799).
                For further information see: https://jeremylong.github.io/DependencyCheck/general/suppression.html
            </p>
        </dd>
    </dl>
